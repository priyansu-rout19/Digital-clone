Here's where we've landed after this whole investigation:
The architecture is sound. The 19-node LangGraph pipeline, the two-tier retrieval, the conditional edges for skipping retrieval on conversational intents, the confidence scoring, the review queue — all of this is well-designed and working as intended. The system is not fundamentally broken. It has two specific deficiencies that create a cascading failure.
Deficiency 1: The intent gate is single-point-of-failure. The LLM classification is the only thing standing between a greeting and the full RAG pipeline. When it misclassifies (and it will — Qwen3.5 at the dev proxy level is not perfectly instruction-following), there's no safety net because the fallback is trapped inside an exception handler. The fix: run regex pre-filtering before the LLM call for obvious cases, and run it after as a validation check for ambiguous cases. The conversational skip-path already exists and works — it just needs reliable activation.
Deficiency 2: The persona is a formatting instruction, not an identity. Even when the pipeline works perfectly — correct intent, good retrieval, accurate citations — the clone sounds like "AI using Parag Khanna's corpus" rather than "Parag Khanna talking to you." The 2-sentence bio and the "You MUST cite your sources" instruction leave no room for personality, warmth, curiosity, or conversational naturalness. The fix: build a rich persona document (identity facts, voice patterns, relational style, boundary rules) and make the generation prompt intent-aware so conversational and substantive responses get different instructions.
Plus three smaller fixes that fall out naturally:

Confidence scoring needs an intent-aware factor so conversational responses aren't penalized for not using retrieval
Mem0 injection needs to move to the system prompt with proper framing
Conversation history formatting needs to separate history from current context


What the SOLVE.md should contain, based on everything we've discussed:
The document should be structured around five changes, ordered by impact and dependency:
S1 — Harden the intent gate (unblocks everything else). Move regex pre-filter before LLM call. Add post-LLM validation. This is the "one fix that would have prevented the screenshot failure."
S2 — Build the persona document (the soul). A structured persona document for Parag Khanna covering identity, voice, relational style, and boundaries. Stored in the clone profile or as a separate file referenced by it. Best-effort from existing corpus transcripts, with a note that it should be refined with the client.
S3 — Make generation intent-aware (graduated modes). The interpretive_generator_prompt() function gets intent_class and persona_document parameters. Conversational messages get "respond naturally, no citations needed." Substantive questions get "cite sources but weave them in conversationally." This is the change that makes even RAG-backed responses feel human.
S4 — Fix confidence scoring (stop flying blind). Add an intent-appropriateness factor. For conversational intents, measure whether the response was warm, brief, and persona-consistent — not whether it cited passages. This fixes the monitoring dashboard so you can actually see quality.
S5 — Fix memory and context injection (polish). Move Mem0 to system prompt, frame it as "you remember this person," separate conversation history from retrieved context in the user message.
